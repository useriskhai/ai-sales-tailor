# テンプレートテスト画面

## 画面概要
テンプレートの現在の設定に基づいてセールスレターを生成するためのテスト実行画面。
設定を一時的に変更してテストを行うことで、異なるアプローチやトーンでのセールスレターを試すことができます。

## データモデル

### テスト設定
```typescript
interface TestConfig {
  // テンプレートの現在の設定
  currentSettings: {
    messageStrategy: {
      mode: 'ai_auto' | 'manual';
      strategy: Strategy;
      toneOfVoice: ToneOfVoice;
      maxLength: number;
      useEmoji: boolean;
    };
    executionSettings: {
      execution_priority: 'speed' | 'balanced' | 'quality';
    };
  };

  // テスト対象情報
  testTarget: {
    company: {
      id: string;
      name: string;
      industry: string;
    };
    product: {
      id: string;
      name: string;
      features: string[];
    };
  };
}
```

## インタラクティブ要素

### 1. 設定の表示と変更
- **現在の設定表示**:
  - 各設定項目をカード形式で表示
  - 設定値の簡潔な表示
  - 設定変更ボタンを各カードに配置

- **設定変更モーダル**:
  - 現在の設定を初期値として表示
  - 直感的な選択UI（ラジオボタン、ドロップダウン）

## バリデーション

### テスト実行前
- 必須項目の入力確認
- 設定値の範囲チェック

### 設定変更時
- 設定の整合性チェック
- 無効な組み合わせの防止
- 変更内容の妥当性確認

## エラーハンドリング

### テスト実行エラー
- API接続エラー
- リソース不足エラー
- タイムアウト
- リトライメカニズム

### データ検証エラー
- 入力値エラー
- フォーマットエラー
- 整合性エラー

## レスポンシブ対応

### デスクトップ
- 2カラムレイアウト
- サーボードショートカット

### タブレット/モバイル
- シングルカラムレイアウト
- タッチ操作最適化
